# First Principles

This project studies professional learning system design, not dental care delivery.

The first domain is Taiwan-Japan cross-border dental care planning because it creates a realistic need to organize goals, timelines, language context, logistics, prior information, uncertainty, and questions for expert review.

Core principles:

- Decision preparation, not decision making.
- Human judgment remains accountable for goals, context, review, and next steps.
- AI organizes information but does not decide diagnosis, treatment, medication, urgency, waiting safety, or travel safety.
- Synthetic examples are required because the repository is not a patient-data environment.
- Local-first architecture keeps the prototype inspectable and prevents accidental data flows.

The system should help prepare better questions for a dentist. It should never imply that a software artifact can replace dentist or medical professional evaluation.
