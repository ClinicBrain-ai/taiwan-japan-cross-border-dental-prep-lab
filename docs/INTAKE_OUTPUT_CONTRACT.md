# Intake Output Contract

## Purpose

The share-ready Markdown files should define what the AI collects and what it outputs. They are intake and output contracts for BYO-AI dental handoff preparation.

The MD is not a healthcare safety policy. The MD is an intake and output contract.

The contract has four layers:

1. Intake fields
2. Source separation labels
3. Output templates
4. Handoff packet schema

Do not create competing names for the same field. The preferred canonical name for professional source wording is `Dentist original statement`.

## Layer 1: Intake Fields

The AI should collect enough information to populate the canonical fields, one question at a time:

- Case purpose
- Intended recipient
- Language / tone preference
- Main concern
- User-reported symptoms
- Symptom timeline
- Urgent warning status
- Dentist original statement
- User interpretation
- Available records
- Missing records
- Quote / treatment plan line items
- Cost / Quality / Continuity considerations
- Evidence-focused analysis trigger
- Handoff packet target

Unanswered fields remain `unknown`, `not provided`, `needs dentist confirmation`, or `not available in current records`. Do not treat unanswered fields as negative findings.

## Layer 2: Source Separation Labels

Use these labels consistently:

| Label | Definition |
| ----- | ---------- |
| `User-reported symptom` | Symptom, concern, timeline, or experience described by the user |
| `Dentist original statement` | Original or near-original wording from a dentist or dental clinic explanation |
| `Clinic document / quote / receipt` | Written clinic material, estimate, receipt, treatment plan, record list, or billing line item |
| `User interpretation` | What the user thinks a dentist statement, document, quote, or symptom means |
| `AI organization` | Structuring, translation support, summarization, or formatting performed by the AI |
| `AI possible explanation direction` | Evidence-focused possibility framing generated by the AI; not final diagnosis |
| `Missing evidence` | Information, records, exam findings, or dentist explanations not available in the current conversation |
| `Dentist-facing question` | Question prepared for a dentist or clinic |
| `Next-AI-agent question` | Continuation question or instruction for another BYO-AI session |

Source separation rules:

- Dentist statements must not be merged with user interpretation.
- Clinic documents / quote / receipt must stay separate from AI reasoning.
- AI possible explanation directions are not final diagnosis.
- Missing evidence must not be treated as negative evidence.
- Unanswered fields must not be filled with guesses.

### Urgent Warning Status

If no urgent warning signs are reported:

```md
## Urgent Warning Status
- Red flags reported: none so far
- Note: continue structured intake
```

If urgent warning signs are reported:

```md
## Urgent Warning Status
- Red flags reported: yes
- Reported signs:
- Suggested workflow route: seek local dental or medical evaluation first; this worksheet can still organize information for that visit.
```

Urgent warning signs are facial swelling, fever, pus, difficulty swallowing, difficulty breathing, trauma, uncontrolled bleeding, or pain that cannot be controlled.

Do not provide final emergency triage. Do not decide whether it is safe to wait or travel. Do not provide medication dosing instructions. Do not continue deep treatment, cost, travel, or timing analysis while red flags are unresolved.

## Layer 4: Handoff Packet Schema

The final handoff packet should follow `docs/HANDOFF_PACKET_SCHEMA.md` conceptually. It should be readable by a dentist and reusable by a next AI agent.

## What the AI Should Ask

- What is the main concern?
- Are there urgent warning signs that should pause normal planning?
- When did symptoms or events occur?
- What exactly did the dentist say or write?
- What does the user think the explanation means?
- Which records are available?
- Which records are missing?
- What quote or plan line items were provided?
- What is unclear about cost, quality / evidence, continuity, follow-up access, language / communication, insurance, timing, or travel burden?
- Is the output target a dentist, the user, or a next AI agent?

## What the AI Should Output

- A structured Markdown handoff packet.
- A source-separated symptom timeline.
- Dentist original statement and user interpretation as separate fields.
- Available records and missing records.
- Quote / treatment plan line items.
- Cost / Quality / Continuity considerations.
- Optional evidence-focused analysis with uncertainty labels.
- Dentist-facing questions.
- Next-AI-agent questions.
- Final dentist-review boundary.

## Minimal Boundary Placement

The boundary should appear in three practical places:

1. During boot: one short sentence that this is dentist-review preparation.
2. During urgent warning check: route warning signs to local urgent dental or medical evaluation before normal planning.
3. At packet end: a final dentist-review boundary.

Avoid repeating the same disclaimer after every answer. Once the boundary is established, the assistant should keep working on intake, organization, missing evidence, and handoff output.
